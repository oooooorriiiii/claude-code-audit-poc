## 1. タイトル（0:20）
こんにちは。「Claude Code、組織で使うと監査ログどこまで取れるの?」という話です。
結論から言うと、**Pro プランでも自分の Laptop だけ**で、入力からツール実行まで全部ローカルに残せます。5分で実演します。

## 2. 課題（0:30）
Claude Code は強力で、bash もファイル編集も MCP も何でもできる。だからこそ
「誰が・何を入力し・どのツールを・どう判断して実行したか」を後から追えるようにしたい。
でも会話を外部 SaaS に送るのは避けたい。**手元で完結**させたい。
実は Claude Code は **OpenTelemetry** で利用状況を出力できるので、それをローカルで受けます。

## 3. 前提: OTel の3種（0:30）
OTel を触ったことがない方向けに前提だけ。監視データは大きく3種類です。
**数値の時系列＝メトリクス**は Prometheus に。**1件1件の出来事＝ログ/イベント**は Loki に。
**1つの処理の流れと所要時間＝トレース**は Tempo に。
これらを標準フォーマットで吐き出す仕組みが OpenTelemetry。
Zabbix・Datadog・ELK を運用された経験があれば「監視データの“送る側”を業界標準にしたもの」と捉えてください。

## 4. アーキテクチャ（0:35）
構成はこれだけ。ホストの Claude Code が OTLP・gRPC 4317 に吐く。
Docker の OTel Collector が受けて、**メトリクスは Prometheus / ログは Loki / トレースは Tempo** に振り分け、
**生 JSON もファイル**に残す。最後に **Grafana** で全部見る。`docker compose up` 一発で立ちます。

## 5. デフォルトで取れるもの（0:25）
まず素の状態。メトリクスでトークン・コスト・セッション数。
イベントで `user_prompt`・`tool_result`・`api_request` など。
各レコードに `user.email` や `session.id` が付くので、**誰がどのツールをいくらで使ったか**はこれだけで分かる。
ただし「中身」（入力本文や Claude の回答）はデフォルトでは伏せられています。

## 6. ★ フラグで中身まで（0:35）★山場
ここが本題。これらの環境変数を開けると—
入力本文、ツールの引数、ツールの入出力、そして **Claude の出力本文・会話履歴の全文**、
さらに **分散トレース**まで取れます。
デフォルト OFF なのはプライバシーのため。裏を返せば、開ければ監査に必要なものはほぼ全部。
では実際に取れているところを見せます。

## 7. デモ: 端末（1:10）★動画再生
（端末デモ動画を再生しながら）
フル監査モードで `claude` を1回叩いています。そのあとローカルに残ったものを順に表示します:
- **① 入力本文** — ユーザが打ったプロンプトそのもの
- **② Claude の出力本文** — `api_response_body` の中の回答テキスト
- **③ tool 入力引数** — 実行された bash コマンドなど
- **④ トレース階層** — `interaction` の下に `llm_request` と `tool→execution` がぶら下がる

全部、手元のファイルとバックエンドに入っています。外には出していません。

## 8. Grafana: GUI ウォークスルー（0:35 / 詳説版は伸長可）
可視化は Grafana。動画で順に解説します。
**①ダッシュボード**で“量”の全体像(トークン・コスト・セッション数)。
**②Loki**でイベントの“中身” — 行を開くと user.email や入力本文まで属性が見える。
**③Tempo**でトレース — 1プロンプトの処理の流れと、ツールのどこで時間がかかったかが分解できる。
**ログの `trace_id` からトレースへ**、相互ジャンプも設定済み。
（詳しく見せる場合は動画を止めつつ `gui-guide.md` の各画面解説を補足）

## 9. まとめ（0:30）
- Laptop と `docker compose` だけで監査基盤が立つ
- テレメトリ単独で **入力・出力・tool 入出力・トレース**まで取れる
- 唯一の限界は **extended thinking（拡張思考）の本文がマスク**されること
- フルモードは**会話全文が残る**ので、保存先・保持期間・アクセス制御はちゃんと設計を

### Q&A 想定
- **Q. Pro でも本当に動く?** A. テレメトリはプラン非依存。env だけで有効。
- **Q. 常時 ON になる?** A. env を渡したプロセスのみ。`claude-audit.sh` / `-full.sh` で実行時限定。
- **Q. traces が出ない** A. `CLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1` + `OTEL_TRACES_EXPORTER=otlp` が必要(ベータ)。
- **Q. 既存 SIEM へ** A. Collector の exporter を増やすだけ(otlphttp / kafka 等)。
